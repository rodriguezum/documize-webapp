Who are the typical users of Documize

With documize we help users that:

Need to protect the view of specific users on the file system/knowledge base

Draft and execute service agreements and statements of work in predefined templates and content blocks in a workflow

Receive document changes in request queues and with status updates

Want to have a Quality Assurance process to make sure that the documents hold all mandatory content blocks

Want to test test different variations of a proposal and track when and how far a user reads and uses/changes the documents

With documize we help users that:

Want to assign/review tasks directly within the document without jumping tools manage multiple versions of the same/similar document

Need to track which users add what document sections and which changes are made

Want to protect sensitive parts of documents like costs from certain users

Need an automated workflow that when a document is partially or fully ready, it gets assigned to the next user

Want protect the document either fully or partially against the view of unwanted users

need to track document work completion against targets

With documize we help users that:

Use tools like Airtable, Trello, GitHub,… and need to keep an updated view of the status in a “live document”

Want to make comments and collaborate directly on the document

Need to track what changes where done since the last time you reviewed the document

Need to track performance and efficiency of how the document is being produced and at what speed

Need content blocks to build documents in a faster way

With documize we help users that:

Need to combine different content such as contract provisions making sure each version is the latest version

Need a simple way review that all content blocks fit together

Need to authorise new versions and automate distribution of documents

Use spaces where published documents gets stored and visible to some users and “in development” versions are stored in other spaced for other people to work on
